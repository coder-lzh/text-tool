实现多人合作程序开发的过程中，我们有时会出现错误提交的情况，此时我们希望能撤销提交操作，让程序回到提交前的样子，本文总结了两种解决方法：reset、revert。

reset	该命令会强行覆盖当前版本和要回退的版本之间的其他版本（不太建议）
revert	再当前版本的基础上新增一个版本，不影响以前的代码


reset 示例：
1、git log查看要回退的版本号
2、使用 git reset --hard （目标版本号）命令将版本回退
3、这样就是回退成功了！再通过git log查看记录，之前的的提交记录都没了，都被覆盖了
4、此时再推到远程仓库用git push 会报错，需要用git push -f强推上去才可以！！！

==================================================================

revert 示例：
1.git log 查看版本号
2.git revert -n (版本号)
